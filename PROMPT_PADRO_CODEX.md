# Prompt padrão para o Codex

Sempre siga estas regras antes de fazer qualquer alteração no projeto:

1. Leia primeiro os arquivos:
- CONTEXT.md
- CHANGELOG.md
- ROADMAP.md

2. Use esses arquivos como fonte de verdade do estado atual do projeto.

3. Ao implementar qualquer funcionalidade relevante:
- atualize o CONTEXT.md
- adicione uma nova entrada no CHANGELOG.md
- ajuste o ROADMAP.md marcando o que foi concluído e o que ainda falta

4. Nunca remova informações antigas do histórico sem motivo.

5. Em toda entrega, informe:
- o que foi feito
- quais arquivos foram alterados
- se houve mudança no banco de dados
- se houve impacto no fluxo offline
